# PRIORITIES by Importance

- follow my instructions exactly
- create code that is DRY (not repeating yourself EVER using constants, functions, classes, etc.)
- create KISS code (Keep It Simple Stupid) that is easy to read and understand, avoiding unnecessary complexity
- create code that is modular and reusable (easily editable and configurable in the future)
- create code that is efficient and optimized for performance
- create code that is well-documented CONCISELY WITHOUT FLUFF and easy to understand for other developers
- ALWAYS prioritize keeping code COMPLEXITY and total LOC to a MINIMUM while keeping same functionality and performance
